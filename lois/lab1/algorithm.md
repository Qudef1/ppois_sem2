1. На вход подается строка с формулой
2. Если строка пустая, возвращается False. 
3. Если первый символ не '(', переходим к 4
4. Выполнение parse_atomic()
4.1. Если текущий элемент не '(', переходим к 5
4.2. Если следующий за текущим элементом '!', переходим к 5.
Алгоритм parse_cnf():
5.1. Если текущий символ не '(', переходим к parse_atomic().
5.2. Если текущий символ '(', вызываем _is_conjunction_bracket() для определения типа операции.
5.3. Если обнаружена конъюнкция '/\' на первом уровне вложенности:
5.3.1. Потребляем открывающую скобку '('.
5.3.2. Рекурсивно вызываем parse_cnf() для левой части.
5.3.3. Проверяем наличие оператора '/\', потребляем его.
5.3.4. Рекурсивно вызываем parse_cnf() для правой части.
5.3.5. Проверяем отсутствие цепочек операций без скобок.
5.3.6. Ожидаем закрывающую скобку ')'.
5.3.7. Возвращаем True при успешном разборе.
5.4. Если конъюнкция не обнаружена — переходим к parse_atomic().
Алгоритм parse_atomic():
6.1. Если текущий символ не '(', переходим к parse_literal().
6.2. Если текущий символ '(', а следующий '!' — переходим к parse_literal().
6.3. Иначе (начало дизъюнкции в скобках):
6.3.1. Потребляем '('.
6.3.2. Рекурсивно вызываем parse_atomic() для левого операнда.
6.3.3. Проверяем наличие оператора дизъюнкции '\/', потребляем его.
6.3.4. Вызываем parse_literal() для правого операнда.
6.3.5. Проверяем отсутствие цепочек операций (A\/B\/C) без дополнительных скобок.
6.3.6. Ожидаем закрывающую скобку ')'.
6.3.7. Возвращаем True при успехе.
Алгоритм parse_literal():
7.1. Если текущий символ '(':
7.1.1. Если следующий символ '!' — это отрицание в скобках:
7.1.1.1. Потребляем '(!'.
7.1.1.2. Проверяем, что далее не идёт '(' (отрицание только к переменной).
7.1.1.3. Вызываем parse_variable().
7.1.1.4. Ожидаем закрывающую скобку ')'.
7.1.2. Иначе — ошибка: литерал в скобках без отрицания недопустим.
7.2. Иначе — вызываем parse_variable() для простого литерала.
Алгоритм parse_variable():
8.1. Проверяем, что текущий символ — заглавная латинская буква (A-Z).
8.2. Если нет — ошибка: некорректное имя переменной.
8.3. Потребляем букву.
8.4. Пока текущий символ — цифра:
8.4.1. Если цифра '0' — ошибка (запрещены ведущие нули в индексах).
8.4.2. Потребляем цифру.
8.5. Возвращаем True при успешном разборе переменной.
Вспомогательный метод _is_conjunction_bracket():
9.1. Сканируем формулу начиная с позиции после текущей '('.
9.2. Отслеживаем уровень вложенности скобок (depth).
9.3. При depth == 1 проверяем двухсимвольные операторы:
9.3.1. Если найден '/\' — возвращаем True (это конъюнкция на верхнем уровне).
9.3.2. Если найден '\/' — возвращаем False (это дизъюнкция, обрабатывается в parse_atomic).
9.4. При достижении закрывающей скобки с depth == 0 завершаем сканирование.
Финальная проверка в методе check():
10.1. Если формула пустая — ошибка.
10.2. Запускаем parse_cnf() в блоке try-except.
10.3. После успешного парсинга проверяем, что указатель достиг конца строки (pos == length).
10.4. Если остались необработанные символы — ошибка: "Парсер не отработал до конца".
10.5. Возвращаем результат проверки и сообщение об ошибке при необходимости.
