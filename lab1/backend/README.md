# Lab 4: Web-интерфейс театра (FastAPI)

## Что реализовано

- Клиент-серверное веб-приложение на `FastAPI + HTML/CSS/JS`.
- Общий доменный код сохранен: веб-слой использует существующие классы из `lab1/src`.
- Перенесены операции из CLI (`src/main_menu.py`) в веб-формы:
  - добавление зала, актера, режиссера, постановки, костюма;
  - привязка постановки к залу и генерация билетов;
  - добавление актера в постановку, назначение костюма;
  - добавление репетиций и отметка актеров;
  - продажа билетов;
  - сохранение и загрузка состояния JSON;
  - изменение названия театра и отображение текущего состояния модели.

## Архитектура

- `app/theater_service.py` — application service (use-case слой), инкапсулирует работу с моделью `Theater`.
- `app/container.py` — простой DI-контейнер с singleton-сервисом и шаблонизатором.
- `app/dependencies.py` — FastAPI dependencies для внедрения сервиса.
- `app/main.py` — web-роуты.
- `app/templates/index.html` — UI с формами.
- `app/static/*` — стили и клиентский JS.

Такое разделение поддерживает принципы SOLID:
- SRP: роуты не содержат бизнес-логику, только orchestration.
- DIP: роуты зависят от абстракции сервиса через dependency function.
- OCP: новые операции добавляются в сервис без переписывания шаблона/ядра.

## Запуск

```bash
cd lab1/backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Открыть в браузере: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## Совместная работа CLI и Web

- CLI продолжает работать как раньше:
  - `cd lab1/src`
  - `python main_menu.py`
- Web использует те же доменные классы (`Theater`, `Setting`, `Actor`, и т.д.) из `lab1/src`.
- За счет этого обе версии интерфейса работают поверх общей предметной модели.
